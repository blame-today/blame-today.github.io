how this works

every vote is a signed nostr event. our app is just one view of the public ledger. cloudflare is a cache. anyone can audit.
vote flow: iOS client signs vote with a per-device nostr keypair (BIP-340 Schnorr) ├─→ publishes signed event to 7 public nostr relays (the ledger) └─→ POSTs to cloudflare worker → D1 rollup (the cache) read path: this page → cloudflare worker → D1 → CDN-cached JSON (fast) auditors → any nostr relay → recompute from events (true) if the two ever disagree, the public ledger wins. the cache is opinion; the relays are the record.

verify the math

we don't ask you to trust us. here's exactly how to recompute today's leaderboard yourself from the public nostr ledger. if your numbers don't match ours, we're lying. call us out.
protocol: nostr NIP-01, kind 30078 (parameterized replaceable) filter: {"kinds":[30078], "#d":["blame:vote:<YYYY-MM-DD>"]} relays we publish to: relay.damus.io nos.lol relay.nostr.band relay.snort.social relay.primal.net offchain.pub nostr.mom steps: 1. connect to any relay above via WebSocket 2. send the filter (see /verify?day=<day> for the exact REQ) 3. for each EVENT, parse content as JSON → (pubkey, target) 4. group by target. unique pubkey count per target = real_count 5. compare to /leaderboard?day=<day> machine-readable: /verify?day=<today>

why this matters: we don't own the data — the protocol does. you don't have to trust the company, the company can't lie undetectably, and if we shut the worker off tomorrow the votes are still there. that's the property we want.
